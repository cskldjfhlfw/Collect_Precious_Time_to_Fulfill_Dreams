# 用户数据文件说明

## 关于 Semgrep 检测到的 bcrypt 哈希

Semgrep 在 `users.csv` 文件中检测到 bcrypt 哈希值。这是**预期行为**，不是安全问题。

### 说明

- `users.csv` 包含测试/演示用户数据
- 密码已经使用 bcrypt 进行了安全哈希处理
- bcrypt 是一种安全的密码哈希算法
- 这些哈希值**不能**被逆向破解为原始密码

### 安全性

✅ **安全的做法**:
- 存储密码的 bcrypt 哈希值（而不是明文密码）
- 使用足够的 salt 和迭代次数
- 哈希值即使泄露也无法还原原始密码

❌ **不安全的做法**:
- 存储明文密码
- 使用弱哈希算法（如 MD5、SHA1）
- 不使用 salt

### Semgrep 警告

Semgrep 检测到 bcrypt 哈希是因为：
1. 它识别出了 bcrypt 哈希的特征模式（`$2b$12$...`）
2. 这是一个信息性警告，提醒你文件中包含密码哈希
3. 这**不是**一个安全漏洞

### 建议

对于生产环境：
1. ✅ 将 `*.csv` 添加到 `.gitignore`（已完成）
2. ✅ 不要将包含用户数据的文件提交到版本控制
3. ✅ 使用数据库存储用户信息
4. ✅ 定期更新用户密码哈希算法

### 忽略此警告

如果你确认这是测试数据且已采取适当的保护措施，可以安全地忽略 Semgrep 关于此文件的警告。
